# GitHub Presentation Polish Design

**Date:** 2026-03-11

## Goal

Improve the public GitHub presentation of `Codex Desktop Manager` without changing product behavior.

## Scope

- remove release-generation instructions from the repository landing pages
- rewrite the first GitHub Release notes in clearer Simplified Chinese
- tidy the repository About panel metadata
- add relevant Topics
- keep Homepage empty for now

## Why

- visitors should land on a concise product page, not a maintainer workflow page
- direct download information belongs on the Release page, not the README home page
- better About and Topics metadata improve discoverability and first impressions

## Expected Result

- Chinese README home page stays focused on what the app does
- Release `v0.1.0` explains which file to download in Chinese
- repository About is concise and accurate
- Homepage remains unset until a future project site exists
