# README Screenshot Design

**Date:** 2026-03-12

## Goal

Add the latest product screenshot to the Chinese and English README landing sections so GitHub visitors can immediately see the current UI.

## Scope

- copy the user-provided screenshot into a repository asset folder
- add one screenshot section to `README.md`
- add one screenshot section to `README.en.md`

## Why

- the project now has a polished UI and direct visual context improves the repository landing page
- keeping the screenshot in-repo makes the README self-contained and stable on GitHub

## Expected Result

- both README files display the same current account-pool screenshot near the top
- the image is stored under a predictable docs asset path inside the repository
